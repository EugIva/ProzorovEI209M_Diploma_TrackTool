### help_sniffer
Вкладка «онлайн-сниффер» позволяет в режиме реального времени сохранять в json-файле воздушную обстановку с информацией по каждому воздушному судну. Здесь возможно задать время работы программы в секундах – это время отслеживания воздушной обстановки, затем задать точку с радиусом, создав таким образом зону отслеживания. Но чем больше размер зоны, тем меньшую точность координат предоставляет API. При этом максимальной точностью координат, снятых таким образом, может быть 4 знака после запятой. Далее пользователь может задать имя файла, в который будет сохранена информация. Дополнительной опцией является фильтр. Настроки фильтра позволяют отслеживать не всё подряд, что залетает в указанную зону, а получать данные или только для указанных номеров рейсов, или отбирать по аэродрому вылета, или основываясь на положении в воздухе или на земле. Таким образом получается уменьшить количество лишней информации в получаемом файле. Затем остаётся нажать на «запустить» и ждать конца выполнения процесса онлайн-сниффера.

### help_trails
Вкладка "слепок" позволяет сохранить трековую информацию с более высокой точностью для всех самолётов, попавших в указанную пользователем зону. Но, в отличие от онлайн-сниффера, данные берутся не в реальном времени для самолётов, а из данных о пройденном пути самолёта. Таким образом, мы получаем трек самолёта, попавшего в нашу зону, за всё время его полёта до текущего момента. Данные координат долготы и широты получаются с максимальной точностью в 6 знаков после запятой. Дополнительным функционалом этого инструмента является возможность увеличить длительность сканирования треков. Можно увеличить количество итераций, и настроить время паузы между ними. Одна итерация получает треки всех самолётов, попавших в область, а следующая – делает то же самое, дополняя уже полученные данные Ещё одной функцией этого инструмента является возможность делать слепки сразу для нескольких разных зон. Пользователю надо занести в таблицу координаты и радиус зоны, и указать паузу между сменой зон. Количество зон не ограничено. После работы над настройками, требуется указать имя вашего сеанса слепка, и запустить программу. О завершении работы программы вам даст знать уведомление на рабочем столе со звуком и отображение информации о проведённом сеансе в нижней части программы. Посмотреть полученный файл можно по кнопке «открыть папку», чтобы сразу оказаться в нужной директории.

### help_excel
Для каждого экспорта создаётся новая папка с именем преобразумеого сеанса. Затем для каждого рейса, который попал в процесс получения треков, создаётся свой Excel файл с именем рейса, содержащий поля: время, идентификатор, имя рейса, название ВС, номер ВС, ИКАО код авиакомпании, статус на земле или в воздухе, скорость наземная (в узлах), вертикальная скорость (в узлах), высота (в футах), долгота, широта, код ИАТА аэродрома вылета и аэродрома назначения.

### help_kim
Функционал вкладки экспорта в JSON для KIM заключается в экспорте полученного результата в другой формат (flow), который можно загрузить в Комплекс Имитационного Моделирования ОрВД для проведения рассчётов с полученным потоком.

### help_drawSpeed
На вкладке отрисовки графиков скоростей требуется указать путь к файлу, и выбрать тип скорости, для которой требуется отобразить скорость – наземную скорость (groundSpeed_Kts), и вертикальную скорость (verticalSpeed). После нажатия на кнопку «отрисовать график скорости» в новом окне создастся график в координатах скорости и времени. Для удобного чтения графиков рекомендуется использовать функцию разделения Json, чтобы строить график для отдельных ВС, а не всего потока сразу. 

### help_flowSorter
Вкладка ручной сортировки потока – ещё один инструмент подготовки данных для обучения нейросети. Пользователь сможет разделить один поток на два файла, сортируя треки на «корректные» и «испорченные». Как на вкладке слияния и редактирования, при загрузке трека(ов), они будут загружены в таблицу. Требуется выбрать один или несколько рейсов, и определить их в какую-нибудь категорию, нажав на кнопки «Корректный» или «Испорченный». Счётчик под таблицей будет отображать, сколько треков было отправлено в каждую категорию, и сколько осталось. Кнопкой «отменить шаг» можно вернуть один последний рейс, определённый в категорию, в общую таблицу, чтобы изменить решение. Кнопка «очистить всё» очистит обе категории от рейсов, которые были в них определены, они станут пустыми.
Когда будет завершена работа по сортировке потока, и после нажатия на «отсортировать» в директории Exports\flowSorted будет создана папка с названием, указанным в соответствующем поле, а в ней – два JSON файла с постфиксами «_Correct» и «_Incorrect». В них будут записи тех рейсов, которые были разделены по категориям. Оставшиеся в таблице рейсы не будут никуда помещены. 

### help_jsonDivide
Разделение JSON удобно применять, когда требуется разделить полученный файл потока на раздельные рейсы, сохранив структуру файла такой, чтобы трек можно было дальше обрабатывать средствами TrackTool. Пользователю надо выбрать разделяемый файл, и запустить программу. В директории Exports\jsonDivide будет создана папка с названием выбранного потока, а внутри будут созданы JSON файлы для каждого рейса. В каждом таком файле будут записи только для одного рейса, позывной которого указан в названии. 

### help_jsonMerge
Данный функционал поможет как объеденить разные файлы потоков в один, так и почистить какой-нибудь файл потока, удалив неподходящие рейсы. Полученный файл потока можно и дальше обрабатывать средствами TrackTool.
На вкладке слияния и редактирования JSON пользователю необходимо загрузить файлы потока (один или несколько). Они будут обработаны и загружены в таблицу. Колонки можно отсортировать для удобства. Каждая строчка представляет один рейс, и в таблице будет отображен путь к файлу, которому принадлежит этот рейс, идентификатор рейса (id), номер рейса (callsign), время полёта (flight_time), количество точек маршрута (route_points), аэропорт вылета (airport_origin), и аэропорт прилёта (airport_destination). Пользователь может просмотреть все записи в таблице, выделить любое количество строк и удалить их с помощью кнопки «удалить». Рейсы пропадут из таблицы, и не попадут в итоговый файл. После работы над потоком, надо ввести название нового файла, и нажать «создать JSON». В директории Exports\jsonMerge будет создан новый JSON файл, в который попадут рейсы, которые были отображены в таблице

### help_flowInfo
На этой вкладке можно посмотреть информацию о потоке – количество рейсов в потоке, есть ли дублирующиеся номера рейсов (когда разные id, но одинаковые callsign), какая максимальная точность координат (количетсво знаков после запятой), диапазон времени (дата и время у самой первой и последней точки маршрутов потока), всего времени (сколько времени в сутках, часах, минутах, секундах). 

### help_flowFilter
Функционал на вкладке авто-фильтрации потока позволяет пользователю быстро обработать файл потока, активировав разные фильтры одной кнопкой, и получив обработанный файл потока, содержащий рейсы, которые попали в критерии фильтров. Таким образом кратно облегачется этап подготовки данных для обучения нейросети.
Для автоматической фильтрации пользователю надо выбрать обратываемый файл, затем выбрать те фильтры, которые необходимы. 

### help_atfm
Функционал вкладки экспорта в JSON для ATFM-model заключается в экспорте полученного результата в другой формат JSON, который можно загрузить в комплекс моделирования процессов организации потоков воздушного движения для проведения рассчётов с полученным потоком.
Конвертация в JSON другого формата требует дополнения файла новыми данными – внутренними значениями под формат программы, и значениями к рейсу – значения категории веса, приоритета, пройденного пути, этапа полёта, и прочего.

### help_draw2D
После нажатия кнопки «отрисовать треки в 2D», появляется новое интерактивное окно с плоскостью, на которую нанесены треки, а так-же точкой на карте указаны координаты аэропорта Шереметьево. В легенду вынесено соответсвие цветов и номеров рейсов. Кроме интерактивного окна, приложение сохраняет полученный график в виде изображения в соответсвующей директории, которую можно открыть по клику «открыть папку».

### help_drawVert
Для создания графика высоты полёта самолёта будут взяты графы времени и высоты полёта самолётов. При генерации треков высота сохраняется в футах, но для удобства пользователя, при построении графика вертикального профиля высота будет преобразована в метры. Пользователь может выбрать файл для преобразования, или вставить в поле ввода скопированный путь с названием файла, затем в новом окне будет создан интерактивный график с высотами полёта.

### help_draw3D
Отрисовка треков в 3D позволяет получить представление о состоянии воздушного пространства, взаимодействии самолётов друг с другом, загруженности пространства. Можно заметить схожесть треков друг с другом, и тем самым определить, где какие эшелоны находятся, какие схемы захода на посадку используются. Для построения такого графика достаточно указать путь до файла с треками.

### help_drawMap
Вкладка «карта» позволяет пользователю отрисовать полученные им треки на реальной карте. При этом возможно выбрать подходящее изображение карты из нескольких вариантов – Open Street Map, Google Map, Google спутник, Google Map с рельефом. Пользователь выбирает файл треков, нажимает кнопку «отобразить», и на карте ниже появляются разноцветные треки воздушных судов. Каждый ВС указан собственным цветом для удобства.
